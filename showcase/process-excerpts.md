# Process Excerpts — Showcase Wall

Pull-quotes from the Residual Motion process journal.
Print as cards (~A5 each) or mount on wall in a 2×2 grid.

**Formatting per card:**
- Header: 14pt bold, sans-serif
- Quote: 12pt regular, sans-serif
- Context note: 10pt italic
- Date: 10pt light/grey

## Card 1 — Tool-Builder, Not Designer

> Instead of prompting for visual changes, I built the tool to make them myself. A frosted-glass slider panel that writes directly to the GPU uniforms in real time. I could explore the parameter space by feel, watching the particles respond as I dragged. I added a copy button that diffs current values against defaults — I paste that directly into chat and the AI applies the changes to the codebase.

*The moment text prompting was replaced by direct manipulation.*

15 March 2026

---

## Card 2 — The Repo as Shared Brain

> The CLAUDE.md isn't just describing the system, it's training the next session's collaborator. There's something fitting about this workflow shift happening in a project about embodied AI interaction. The thesis is that the body can provide continuous, unprompted context to an AI. The CLAUDE.md does something analogous: the repository provides continuous, structured context to the AI without me having to narrate it each time.

*The repository mirrors the project's own thesis about continuous context.*

16 March 2026

---

## Card 3 — Asking, Not Generating

> Instead of diving into implementation, I used Claude to ask me questions, treating it as a design crit partner rather than a code generator. Claude asked things like: "Should the exercise structure remain visible or become ambient?" These weren't questions I'd asked myself yet. The structured format forced decisions I'd been deferring.

*AI as interrogator of assumptions, not producer of solutions.*

8 March 2026

---

## Card 4 — The Body Is the Only Prompt

> The user never types. The body is the only prompt. This sounds obvious given the project thesis, but it required actively resisting the chatbot default. Removing the text box keeps attention on the body, not the screen.

*The design decision that defines the entire interaction model.*

17 February 2026
